Learning Objectives

By the end of this lesson, you will be able to:

    Customize FastAPI's auto-generated Swagger UI and ReDoc documentation
    Add descriptions, examples, and tags to make documentation clear and useful
    Define response models and error responses in the docs
    Explain the OpenAPI specification and how FastAPI generates it

You've been using Swagger UI throughout this module — clicking "Try it out," testing endpoints, checking response formats. But you may not have realized that this documentation is generated entirely from your code. Every type hint, Pydantic model, router tag, and docstring you write becomes part of the documentation automatically.


This is one of FastAPI's killer features: your documentation is always accurate because it's always generated from the source of truth — your code. In this lesson, you'll learn to make that auto-generated documentation truly excellent.


Think of it like the difference between a house with a floor plan and a house without one. Both houses exist, but only one lets someone new understand the layout without walking through every room. Good API documentation is the floor plan that lets other developers understand your API quickly.
The OpenAPI Specification

Behind Swagger UI and ReDoc is the OpenAPI specification — a JSON file that describes your entire API in a standardized format. FastAPI generates this file automatically at /openapi.json.

This means your API documentation is portable — you can export the OpenAPI spec and import it into Postman, generate client libraries, or publish it on a documentation site.

Tags for Organization
tags_metadata = [
    {"name": "Authentication", "description": "Register, login, and manage tokens"},
    {"name": "Tasks", "description": "Create and manage tasks"},
    {"name": "Users", "description": "User profile management"},
]

app = FastAPI(title="Task Manager", openapi_tags=tags_metadata)

Response Examples
class TaskResponse(BaseModel):
    id: int
    title: str
    priority: str
    completed: bool

    class Config:
        from_attributes = True
        json_schema_extra = {
            "example": {
                "id": 1,
                "title": "Write documentation",
                "priority": "high",
                "completed": False
            }
        }
